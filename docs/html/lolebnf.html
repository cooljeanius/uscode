The LOLCODE EBNF

Presented below is the EBNF (see http://en.wikipedia.org/wiki/Extended_Backus%E2%80%93Naur_Form) for LOLCODE that lci parses. Note that by this stage, the scanner has:

  • already removed any whitespace between tokens,
  • added in and truncated newline tokens at logical line breaks, and
  • added an end-of-file ($) token.

Program Structure

These production rules dictate the overall form of the program.

MainNode ::= TT_HAI version TT_NEWLINE BlockNode $
BlockNode ::= StmtNode *

Statements

These production rules specify some general types of parse structures.

ConstantNode ::= Boolean | Integer | Float | String
IdentifierNode ::= Identifier
TypeNode ::= TT_NOOB | TT_TROOF | TT_NUMBR | TT_NUMBAR | TT_YARN

Statements

These production rules specify the types of statements formed.

StmtNode ::= CastStmtNode | PrintStmtNode | InputStmtNode | AssignmentStmtNode | DeclarationStmtNode | IfThenElseStmtNode | SwitchStmtNode | BreakStmt | ReturnStmtNode | LoopStmtNode | FuncDefStmtNode | ExprStmt
CastStmtNode ::= IdentifierNode TT_ISNOWA TypeNode TT_NEWLINE
PrintStmtNode ::= TT_VISIBLE ExprNodeList [ TT_BANG ] TT_NEWLINE
InputStmtNode ::= TT_GIMMEH IdentifierNode TT_NEWLINE
AssignmentStmtNode ::= IdentifierNode TT_R ExprNode TT_NEWLINE
DeclarationStmtNode ::= IdentifierNode TT_HASA IdentifierNode [ Initialization ] TT_NEWLINE
Initialization ::= TT_ITZ ExprNode
IfThenElseStmtNode ::= TT_ORLY TT_NEWLINE TT_YARLY TT_NEWLINE BlockNode ElseIf * [ Else ] TT_OIC TT_NEWLINE
ElseIf ::= TT_MEBBE ExprNode TT_NEWLINE BlockNode
Else ::= TT_NOWAI TT_NEWLINE BlockNode
SwitchStmtNode ::= TT_WTF TT_NEWLINE Case + [ DefaultCase ] TT_OIC TT_NEWLINE
Case ::= TT_OMG ExprNode TT_NEWLINE BlockNode
DefaultCase ::= TT_OMGWTF TT_NEWLINE BlockNode
BreakStmt ::= TT_GTFO TT_NEWLINE
ReturnStmtNode ::= TT_FOUNDYR ExprNode TT_NEWLINE
LoopStmtNode ::= TT_IMINYR IdentifierNode [ LoopUpdate ] [ LoopGuard ] TT_NEWLINE TT_IMOUTTAYR IdentifierNode TT_NEWLINE
LoopUpdate ::= LoopUpdateOp TT_YR IdentifierNode
LoopUpdateOp ::= TT_UPPIN | TT_NERFIN | UnaryFunction
UnaryFunction ::= The name of a previously defined unary function.
LoopGuard ::= TT_TIL ExprNode | TT_WILE ExprNode
FuncDefStmtNode ::= TT_HOWDUZ IdentifierNode IdentifierNode [ FunctionDefArgs ] TT_NEWLINE BlockNode TT_IFUSAYSO TT_NEWLINE
FunctionDefArgs ::= TT_YR IdentifierNode FunctionDefArg *
FunctionDefArg ::= TT_ANYR IdentifierNode
ExprStmt ::= ExprNode TT_NEWLINE

Expressions

These production rules specify the types of expressions formed.

ExprNode ::= CastExprNode | ConstantNode | IdentifierNode | FuncCallExprNode | OpExprNode | ImplicitVariable
CastExprNode ::= TT_MAEK ExprNode TT_A TypeNode
FuncCallExprNode ::= IdentifierNode
OpExprNode ::= UnaryOp | BinaryOp | NaryOp
UnaryOp ::= UnaryOpType ExprNode
UnaryOpType ::= TT_NOT
BinaryOp ::= BinaryOpType ExprNode [ TT_AN ] ExprNode
BinaryOpType ::= TT_SUMOF | TT_DIFFOF | TT_PRODUKTOF | TT_QUOSHUNTOF | TT_MODOF | BIGGROF | SMALLROF | TT_BOTHOF | TT_EITHEROF | TT_WONOF
NaryOp ::= NaryOpType NaryOpArgs TT_MKAY
NaryOpType ::= TT_ALLOF | TT_ANYOF
NaryOpArgs ::= ExprNode NaryOpArg +
NaryOpArg ::= [ TT_AN ] ExprNode
ImplicitVariable ::= TT_IT
